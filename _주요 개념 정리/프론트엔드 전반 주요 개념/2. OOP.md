# 2. OOP

: **객체지향 프로그래밍 (Object Oriented Programming)**

```
프로그래밍에서 
필요한 데이터 추상화 해서 
상태와 행위 가지게 만들고 
그 객체들 간 
유기적 상호작용해 
로직 구성하는 프로그래밍 방법
```

<br>

#### [객체 지향 프로그래밍의 장단점]

---

- **장점**

  ```
  - 코드 재사용 용의
  - 유지 보수가 쉬움 (수정해야할 부분 분리되어 있음)
  ```

- **단점**

  ```
  - 설계시 많은 시간과 노력이 필요
  - 느린 처리 속도, 큰 용량
  ```
